## Design Context

### Users
General audience — atproto/Bluesky-adjacent but not gated to power users. Primary jobs: browsing events, creating events. Likely social/community features over time (following organizers, discussions, etc.).

### Brand Personality
Playful, social, community-first. Warm but not childish. Should feel human, not corporate.

### Aesthetic Direction
- **Not** corporate-SaaS-Meetup (avoid generic card grids, stock "connect with people" energy).
- **Not** gen-z party-flyer maximalism either — no neon chaos, no chaotic type mixing for its own sake.
- Sits between: inviting, editorial-leaning, with personality in the details.
- Theme system: built on `@foxui` with a **base + accent** color model. Users can set their own base + accent for their event pages. Keep this simple two-color contract — don't introduce palettes that break it.
- Light+dark both supported via `mode-watcher`. Default "mist cyan" variant.
- Focus energy on **typography, layout rhythm, and micro-details** rather than sweeping color overhauls.

### Design Principles
1. **Two-color discipline** — every design decision must work with an arbitrary base + accent pair. No hard-coded third/fourth accents.
2. **Warmth through type, not color** — personality lives in font choice, hierarchy, and spacing, not in gradients or glows.
3. **Community feel, not corporate** — avoid stock SaaS tropes (hero metric layouts, identical card grids, polished-but-soulless cards).
4. **Simple contracts** — prefer fewer knobs. If a page-level theme only needs 2 colors, don't sneak in a third.
5. **Editorial over dashboard** — lean into asymmetry, generous whitespace, and readable long text rather than dense UI chrome.
